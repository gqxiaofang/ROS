顶升车型车体使用说明文档

1、软体环境

使用ubuntu 16 + ROS kinetic环境

环境依赖

项目编译前需要先解决依赖问题。

  1. ROS环境安装
  2. can 驱动
# 将代码can_driver/hardware_driver/libusbcan.so 移动到 /lib 下
$ sudo cp can_driver/hardware_driver/libusbcan.so /lib
$ sudo cp can_driver/hardware_driver/libcontrolcan.so /lib
$ sudo cp can_driver/hardware_driver/99-myusb.rules /etc/udev/rules.d/
$ sudo cp can_driver/hardware_driver/usbcan.rules /etc/udev/rules.d/
$ sudo udevadm control --reload

2、功能使用

机体节点的使用

安装如下ros package：

sudo dpkg -i  ros-kinetic-cotek-cmake_3.2.0-0xenial_amd64.deb
sudo dpkg -i  ros-kinetic-cotek-common_0.0.0-0xenial_amd64.deb
sudo dpkg -i  ros-kinetic-cotek-embedded_0.0.0-0xenial_amd64.deb
sudo dpkg -i  ros-kinetic-cotek-msgs_0.0.0-0xenial_amd64.deb

启动车体节点：

roslaunch cotek_embedded cotek_embedded.launch

导航激光的使用

首先需要安装导航激光驱动：

mkdir -p sensor_ws/src
cp -r lidar_driver/pepperl_fuchs_r2000/* sensor/src/
cd sensor_ws
catkin_make install

根据需要修改导航激光launch：

rosed cotek_common dppf.launch

其内容如下：

 <launch>
  <node pkg="pepperl_fuchs_r2000" type="r2000_node" name="r2000_node_1" respawn="true">
    <param name="scanner_ip" value="192.168.0.20"/>
    <param name="frame_id" value="laser1"/>
    <param name="scan_frequency" value="20"/>
    <param name="samples_per_scan" value="4200"/>
    <param name="angle_min" value="-2.3562"/>
    <param name="angle_max" value="2.3562"/>
    <remap from="scan" to="scan_1" />
  </node>

  <node pkg="pepperl_fuchs_r2000" type="r2000_node" name="r2000_node_2" respawn="true">
    <param name="scanner_ip" value="192.168.0.21"/>
    <param name="frame_id" value="laser2"/>
    <param name="scan_frequency" value="20"/>
    <param name="samples_per_scan" value="4200"/>
    <param name="angle_min" value="-2.3562"/>
    <param name="angle_max" value="2.3562"/>
    <remap from="scan" to="scan_2" />
  </node>

</launch>

运行导航激光节点：

roslaunch cotek_common dppf.launch

传感器信息获取

这部分主要直接将各个传感器的数据发布。
发布topic为device_table中定义的各个child_name,所以可以根据device_table中的各个child_name查看各传感器数据信息。
比如device_table中定义了如下四个电机，

        {
            "name": "syntron_motor",
            "install_device": [
                {
                    "child_name": "syntron_left_move_motor",
                    "device_param": {
                        "node_id": 13,
                        "channel": 1
                    }
                },
                {
                    "child_name": "syntron_right_move_motor",
                    "device_param": {
                        "node_id": 14,
                        "channel": 1
                    }
                },
                {
                    "child_name": "syntron_lift_motor",
                    "device_param": {
                        "node_id": 15,
                        "channel": 1
                    }
                },
                {
                    "child_name": "syntron_rotate_motor",
                    "device_param": {
                        "node_id": 17,
                        "channel": 1
                    }
                }
            ]
        }

那么就可以根据child_name:syntron_left_move_motor、syntron_right_move_motor、syntr、on_lift_motor、syntron_rotate_motor获取这四个电机的数据信息。比如可以运行下面命令查看左轮电机数据：

rostopic echo /syntron_left_move_motor

又比如device_table中定义了如下IO：

        {
            "name": "io",
            "install_device": [
                {
                    "child_name": "io",
                    "device_param": {
                        "node_id": 1,
                        "channel": 1
                    }
                },
                {
                    "child_name": "io2",
                    "device_param": {
                        "node_id": 2,
                        "channel": 1
                    }
                }
            ]
        }

那么就可以用下面命令分别查看两个IO的状态：

rostopic echo /io
rostopic echo /io2

机体动作

车体主要提供以下原子操作：托盘抬升、托盘下降、托盘旋转、托盘归零、充电、低功耗模式、enable电机、disable电机
audio部分主要提供遇到特殊情况下的声音报警，可根据需要自行设置
三色灯主要提供运行中状态的展示，可根据需要自行设置
订阅topic：

/jack_up_action

消息内容：
消息类型为cotek_msgs/jack_up_action
